train_auto <- read.csv("AutoTrain.csv",header=TRUE)
test_auto <- read.csv("AutoTest.csv",header=TRUE)
KNN <- function(k,x.train,y.train,x.pred){
#initialize
n.pred <- length(x.pred)
y.pred <- numeric(n.pred)
#main loop
for (i in 1:n.pred) {
  d <- abs(x.train - x.pred[i])
  #dstar <-d[order(d)[k]]
  dstar <- sort(d)[k]
  y.pred[i] <- mean(y.train[d <= dstar])
}
## Return the vector of predictions
invisible(y.pred)
}
library(Metrics)
test_mse_k <- c()
train_mse_k <- c()
k <- c(2,5,10,20,30,50,100)
k.predic <- length(k)
for (i in 1:k.predic) {
   prediction_test <- KNN(k[i],train_auto$weight,train_auto$mpg,test_auto$weight)
   test_mse_k[i] <- mse(test_auto$mpg,prediction_test)
   prediction_train <- KNN(k[i],train_auto$weight,train_auto$mpg,train_auto$weight)
   train_mse_k[i] <- mse(train_auto$mpg,prediction_train)}
mse_train <- data.frame(k,train_mse_k)
mse_train

Comments:

This KNN model looks generally good by setting k=30, so that 30 nearby points are averaged together for prediction yield a best KNN regression. This best KNN model traces the downward slope of the data reasonably well. But the predictions start to get worse when it gets closer to the edges of the data set.

Defining a neighbourhood for KNN regression is to make a good balance between bias and variance:

\(\bullet\) If we choose k too large, we get large testing and training errors(under-fitting the training data). That implies low flexibility, high bias but low variance.

\(\bullet\) If we choose k too small, we get low training and high training erros(over-fitting the training data). That implies high flexibility, low bias but high variance.
